# MemoryMesh — Setup Guide

This guide walks you through deploying and configuring MemoryMesh end to end: AWS infrastructure, Lambda functions, Chrome extension, and Claude Desktop MCP server.

---

## Prerequisites

Before starting, ensure you have:

- **Node.js 20+** — `node --version`
- **npm** — comes with Node.js
- **AWS CLI** — [install guide](https://docs.aws.amazon.com/cli/latest/userguide/install-cliv2.html), configured for `eu-west-2`
- **AWS CDK** — `npm install -g aws-cdk`
- **AWS account** with the following services enabled:
  - Lambda
  - DynamoDB
  - API Gateway
  - Amazon Bedrock (Claude Haiku model access — request in the AWS console under Bedrock → Model access)
  - IAM

Verify your AWS CLI is working:

```bash
aws sts get-caller-identity --region eu-west-2
```

---

## Step 1 — Clone and Install

```bash
git clone https://github.com/yorliabdulai/contextbridge.git
cd contextbridge
npm install
```

This installs dependencies for all packages via npm workspaces.

---

## Step 2 — Deploy AWS Infrastructure

MemoryMesh uses three CDK stacks deployed in order:

| Stack                  | Services                  | What it creates |
|------------------------|--------------------------|-----------------|
| **`MemoryMeshDynamoDB`** | **Amazon DynamoDB**     | `memorymesh-context` table (PK: `userId`, SK: `createdAt`) and `memorymesh-profile` table (PK: `userId`) |
| **`MemoryMeshLambda`**  | **AWS Lambda**, **IAM**, **Amazon Bedrock** | 5 Lambda functions + IAM role with `bedrock:InvokeModel` permissions |
| **`MemoryMeshApi`**     | **Amazon API Gateway**  | HTTP API with routes wired to each Lambda function |

```bash
cd packages/infrastructure
npm install
cdk bootstrap   # Only needed once per AWS account/region
cdk deploy --all
```

CDK will prompt for confirmation before creating resources. Type `y` to proceed.

When complete, copy the **API Gateway URL** from the CDK output — it looks like:

```
MemoryMeshApi.ApiUrl = https://xxxxxxxxxx.execute-api.eu-west-2.amazonaws.com
```

## Step 3 — Package and Deploy Lambda Functions

The Lambda handlers live in `packages/mcp-server/src/lambda/`. After building, they are packaged into a ZIP and deployed to each function.

### Build

```bash
cd packages/mcp-server
npm run build
```

### Package (PowerShell — Windows)

```powershell
Remove-Item lambda-package.zip -ErrorAction SilentlyContinue
Copy-Item package.json dist\
cd dist
npm install --production
cd ..
Compress-Archive -Path ".\dist\*" -DestinationPath ".\lambda-package.zip" -Force
```

### Package (bash — macOS/Linux)

```bash
cp package.json dist/
cd dist && npm install --production && cd ..
zip -r lambda-package.zip dist/
```

### Deploy to all 5 functions

```bash
aws lambda update-function-code --function-name memorymesh-get-context \
  --region eu-west-2 --zip-file fileb://lambda-package.zip

aws lambda update-function-code --function-name memorymesh-save-context \
  --region eu-west-2 --zip-file fileb://lambda-package.zip

aws lambda update-function-code --function-name memorymesh-search-memory \
  --region eu-west-2 --zip-file fileb://lambda-package.zip

aws lambda update-function-code --function-name memorymesh-get-user-profile \
  --region eu-west-2 --zip-file fileb://lambda-package.zip

aws lambda update-function-code --function-name memorymesh-summarize \
  --region eu-west-2 --zip-file fileb://lambda-package.zip
```

### Verify

Test the API is live:

```bash
curl https://YOUR-API-URL.execute-api.eu-west-2.amazonaws.com/profile/test-user
```

You should receive a JSON profile object.

## Step 4 — Install the Chrome Extension

```bash
cd packages/extension
npm install
npm run build
```

This produces a `dist/` folder inside `packages/extension/`.

**Load in Chrome:**

1. Navigate to `chrome://extensions`
2. Enable **Developer mode** (toggle, top right)
3. Click **Load unpacked**
4. Select `packages/extension/dist/`
5. The MemoryMesh icon (⬡) will appear in your toolbar

**Update the API base URL:**

Open `packages/extension/src/utils/api.ts` and ensure `API_BASE` matches your deployed API Gateway URL:

```typescript
const API_BASE = 'https://YOUR-API-URL.execute-api.eu-west-2.amazonaws.com';
```

Rebuild after changing:

```bash
npm run build
```

Then reload the extension in `chrome://extensions` (click the refresh icon on the MemoryMesh card).

## Step 5 — Configure the MCP Server (Claude Desktop)

### Find your config file

- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

### Get your user ID

Your user ID is generated by the Chrome extension on first install. Open the extension popup and copy the UUID shown, or check `chrome.storage.local` in the extension's background page DevTools.

Alternatively, generate a UUID manually:

```bash
node -e "const {randomUUID}=require('crypto'); console.log('mm-'+randomUUID())"
```

### Edit the config

```json
{
  "mcpServers": {
    "memorymesh": {
      "command": "node",
      "args": ["C:\\path\\to\\memorymesh\\packages\\mcp-server\\dist\\index.js"],
      "env": {
        "AWS_REGION": "eu-west-2",
        "CONTEXT_TABLE": "memorymesh-context",
        "PROFILE_TABLE": "memorymesh-profile",
        "MEMORYMESH_USER_ID": "mm-your-uuid-here",
        "AWS_ACCESS_KEY_ID": "your-access-key-id",
        "AWS_SECRET_ACCESS_KEY": "your-secret-access-key"
      }
    }
  }
}
```

**Important:** Use the same UUID in both the MCP server config and the Chrome extension so context saved from the browser is accessible from Claude Desktop.

### Restart Claude Desktop

Fully quit and relaunch Claude Desktop. You should see "memorymesh" listed under connected MCP servers.

### Test the MCP connection

Ask Claude:

> "Use MemoryMesh to get my recent context."

Claude should call `get_context` and return your stored entries.

## Step 6 — Import AI History (Optional)

To bulk-import months of ChatGPT or Claude conversation history:

### Export your data

**ChatGPT:** Settings → Data Controls → Export Data → request ZIP via email  
**Claude:** Settings → Privacy → Export Data → download `data.zip`

### Run the importer

1. Click the MemoryMesh extension icon
2. Click **⬆ Import AI History**
3. Drop your ZIP file onto the upload area
4. The importer auto-detects the format (ChatGPT or Claude) and processes each conversation
5. Each conversation is sent to Bedrock for summarisation, then stored in DynamoDB
6. Progress is shown in real time

> **Note:** The importer uses a 300ms throttle between API calls to avoid hitting Bedrock rate limits. A large export (58+ conversations) takes a few minutes.

## Using MemoryMesh

### In the browser (Chrome extension)

The extension injects a floating banner into Claude.ai, ChatGPT, and Gemini with two buttons:

- **Save Context** — extracts the current conversation, sends it to Bedrock for summarisation, and stores the structured result in DynamoDB
- **Sync to AI** — fetches all stored context entries (up to 1000) and injects the summaries into the chat input, giving the AI instant awareness of your history

The popup shows your 5 most recent context entries and links to the importer.

### In Claude Desktop (MCP tools)

Claude Desktop has access to four tools:

| Tool | Description |
|---|---|
| `save_context` | Store a new context entry |
| `get_context` | Retrieve recent stored entries |
| `search_memory` | Keyword search across all context |
| `get_user_profile` | Get or create your user profile |

## Troubleshooting

**Lambda deploy fails with "Function not found"**  
Ensure CDK has finished deploying all stacks before running the Lambda update commands. The functions must exist before code can be uploaded.

**Bedrock returns errors**  
Confirm Claude Haiku model access is enabled in the AWS console: Bedrock → Model access → `eu.anthropic.claude-haiku-4-5-20251001-v1:0`. The EU cross-region inference profile must be accessible from `eu-west-2`.

**Chrome extension can't reach API**  
Check that `API_BASE` in `api.ts` matches your actual CDK-deployed URL. Rebuild and reload the extension after any change.

**MCP server not appearing in Claude Desktop**  
- Verify the `args` path is absolute and correct for your system
- Ensure Claude Desktop was fully restarted (not just the window — quit from the system tray/menu bar)
- Check that `dist/index.js` exists by running `npm run build` in `packages/mcp-server`

**Context not syncing between browser and Claude Desktop**  
Ensure the `MEMORYMESH_USER_ID` in the MCP config matches the UUID stored in the Chrome extension. Both must use the same user ID to read/write the same DynamoDB partition.
